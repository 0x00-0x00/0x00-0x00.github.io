# Bypass the Block

In my previous AMSI bypass post, I have detailed how AMSI blocks powershell code: Banned strings.

As shown in that same blog post, I detailed some techniques to get around string detection. After some minutes trying to find which word was blocked, I found out that it is something in my Base64 string. So, what to do now?

One thought was to encrypt it with a XOR key and then decrypt it runtime, so we wouldn't use that banned base64 string again in the code. But this seemed over-kill. I wanted to joke as much as Base64 string banning.

Easy. Forget about base64. Its easy to create signatures from that. Think something that is more common and less likely to be banned by AMSI.

My first thought was: Integers.

Convert that base64 string back to binary, and make each byte to become a integer. This would result in a byte array, in integer format.

![Screenshot](/assets/bypass_02.JPG)

Banning sequences of integers is plausible to block this once again, but more likely to cause false positive to a legitimate script, thus, some caution must be applied here, if they want to ban my script again.
